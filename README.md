# Shared Parking Calculator

A Python tool for calculating shared parking demand across multiple land uses using time-of-day adjustment factors. The calculator processes input spreadsheets and generates Excel workbooks with detailed parking analysis.

---

## Project Structure

SharedParkingCalculator/
├── calculate_shared_parking.py   # Main calculation script
├── Inputs/                       # Input Excel files (land-use data & time-of-day factors)
├── Outputs/                      # Generated Excel workbooks (git-ignored)
├── requirements.txt              # Python dependencies
├── uv.lock                       # Deterministic lock file (generated by uv)
├── .gitignore                    # Python-focused ignore rules
├── .vscode/
│   └── settings.json             # VS Code interpreter & formatting config
└── README.md                     # This file

## Environment Layout

The virtual environment lives **outside** the project tree to keep the repo clean and avoid OneDrive sync issues:

| Component               | Path                                                                                              |
|--------------------------|---------------------------------------------------------------------------------------------------|
| **Project root**         | `C:\Users\Mike\OneDrive\Documents\GitHub\Shared Parking Calculator-Clean\SharedParkingCalculator` |
| **Virtual environment**  | `C:\Envs\SharedParkingCalculator`                                                                 |

> **Why external?** Placing the venv outside OneDrive prevents thousands of small files from syncing and avoids path-length issues on Windows.

## Setup

### 1. Create the virtual environment (one-time)

    uv venv C:\Envs\SharedParkingCalculator --python 3.12

### 2. Activate the environment

    C:\Envs\SharedParkingCalculator\Scripts\Activate.ps1

### 3. Install dependencies

    uv pip install -r requirements.txt

### 4. Lock dependencies for reproducibility

    uv pip compile requirements.txt -o uv.lock

This generates `uv.lock` — a pinned snapshot of every package and version. Commit this file to Git so collaborators get identical installs:

    # Collaborators restore the exact environment with:
    uv pip install -r uv.lock

## Usage

### Running the Calculator

    # Make sure the venv is active, then from the project root:
    python calculate_shared_parking.py --dir .


The script reads input files from the `Inputs/` folder, applies time-of-day adjustment factors, and writes results to the `Outputs/` folder.

### Input / Output Workflow

1. **Place input files** in `Inputs/` — CSV files defining land uses, parking rates, and time-of-day factors.
2. **Run the script** — pass `--dir .` (current directory) or a full path to the project root.
3. **Collect results** from `Outputs/` — `WeekdayParking.xlsx` and `WeekendParking.xlsx`.

> **Note:** `Outputs/*.xlsx` files are git-ignored. Regenerate them locally by running the script.

## VS Code Configuration

The `.vscode/settings.json` file is pre-configured to:

- Point the Python interpreter to `C:\Envs\SharedParkingCalculator\Scripts\python.exe`
- Auto-activate the environment in the integrated terminal
- Enable basic type checking
- Format on save with Black (install the **Black Formatter** extension)

If VS Code doesn't pick up the interpreter automatically:

1. Open the Command Palette (`Ctrl+Shift+P`)
2. Select **Python: Select Interpreter**
3. Choose **Enter interpreter path…**
4. Paste: `C:\Envs\SharedParkingCalculator\Scripts\python.exe`

## Troubleshooting

| Problem                            | Fix                                                                 |
|-------------------------------------|---------------------------------------------------------------------|
| `python` not found                  | Activate the venv first: `C:\Envs\SharedParkingCalculator\Scripts\Activate.ps1` |
| VS Code uses wrong interpreter      | Ctrl+Shift+P → Python: Select Interpreter → enter the venv path    |
| `uv` not recognized                | Install uv: `pip install uv` or `winget install astral-sh.uv`      |
| Import errors after fresh clone     | Run `uv pip install -r uv.lock` to restore pinned dependencies     |
| OneDrive sync conflicts             | The venv is external by design — never create a venv inside the repo |
